# 奇怪的国际象棋

## 题目描述

Igor 已经玩国际象棋很久了，现在他厌倦了普通的规则。他要想出新的游戏规则，并成为世界名人。

Igor 的棋盘是一个大小为 $ n×n $ 个单元格的正方形。Igor 认为简单的规则可以保证成功，这就是为什么他的游戏只有一种棋子。此外，他游戏中的所有棋子都是相同颜色的。棋子的可能走法由一组移位向量描述。下一段包含可用走法的正式描述。

让棋盘的行从上到下编号，列从左到右编号为 1 到 $ n $ 。让我们为每个方块分配一对整数 $ (x,y) $ — 对应列和行的编号。棋子的每个可能走法由一对整数 $ (dx,dy) $ 定义；使用这一步，棋子从区域 $ (x,y) $ 移动到区域 $ (x+dx,y+dy) $ 。如果单元格 $ (x+dx,y+dy) $ 在棋盘边界内且不包含其他棋子，则可以执行此移动。在考虑执行给定移动的可能性时，站在除 $ (x,y) $ 和 $ (x+dx,y+dy) $ 之外的单元格上的棋子并不重要（例如，像普通国际象棋中骑士移动时一样）。

Igor 让你找出他的棋子可以做出哪些移动。他在棋盘上放置了几枚棋子，并告诉你每个未占用的方格是否受到任何现有棋子的攻击（即，区域上的某些棋子是否可以移动到该单元格）。恢复棋子的一组可能的移位向量，否则确定 Igor 犯了一个错误，并且这种情况对于任何一组移位向量来说都是不可能发生的。

## 输入格式

第一行包含一个整数 $ n $ ( $ 1<=n<=50 $ )。

接下来的 $ n $ 行包含 $ n $ 个字符，每个字符描述 Igor 提供的位置。第 $ i $ 个字符串的第 $ j $ 个字符可以具有以下值：

- o — 在这种情况下，区域 $ (i,j) $ 被一个棋子占据，并且该区域可能被其他棋子攻击，也可能不被攻击；
- x — 在这种情况下，区域 $ (i,j) $ 被某个棋子攻击；
- . — 在这种情况下，区域 $ (i,j) $ 未被任何棋子攻击。

保证棋盘上至少有一个棋子。

## 输出格式

如果有一组有效的移动，则在第一行打印一个单词“YES”（不带引号）。接下来，以 $(2n-1)×(2n-1)$ 棋盘的形式打印棋子移动集的描述，棋盘中心有一个棋子，符号“x”标记被它攻击的单元格，格式与输入类似。请参阅输出示例以全面了解格式。如果有多个可能的答案，请打印其中任何一个。

如果不存在有效的移动集，则打印一个单词“NO”。

## 提示
在第一个示例测试中，棋子是一个普通的国际象棋棋子好吧，在第二个样本测试中，棋子是一个普通的国际象棋马。

# 题解

我们不要试图找出棋子可以走哪里，而是要找出它不能走哪里。首先将所有的移动标记为可能的；如果有一个区域 (x1, y1) 包含一个棋子，而有一个区域 (x2, y2) 不包含棋子且未被攻击，那么显然移动 (x2 - x1, y2 - y1) 是不可能的。让我们遍历所有棋子和所有未被攻击的区域，并将相应的移动标记为不可能。

假设我们让我们的棋子完成所有其余的移动（尚未标记为不可能的移动），并在所有棋子位于相同位置的情况下重新创建该位置。如果一个区域在初始位置未被攻击，那么在新创建的位置也不会被攻击：事实上，我们已经小心地移除了所有可以将棋子带到这个区域的移动。因此，新位置唯一可能出现的问题是，之前被攻击的某些区域现在未被攻击。但是我们的走法组合是最大的，因为添加任何其他走法都会导致位置不正确。因此，如果新位置与初始位置不一致，则无法重建。否则，我们已经获得了一组正确的走法。此解决方案的复杂度为 O(n4)，用于迭代所有棋子和不受攻击的场地。此过程无需优化即可解决问题。

CF538D题目的核心在于判断给定的棋盘上棋子的移动规则是否合理，并推导出一个可能的棋子移动模式。我们可以通过反向思考棋子不能走到哪些位置，来逐步推导出可能的移动方式。

### 题解过程分块讲解

1. **输入解析和初始化**：
   - 读取棋盘大小`N`和棋盘状态`f`。
   - 创建两个辅助数组`d`和`t`。`d`用来记录哪些移动是可能的，`t`用来模拟生成的攻击范围。
   
   ```cpp
   int N;
   cin >> N;
   vector<string> f(N);
   forn(i, N) cin >> f[i];
   forn(i, 2 * N - 1) forn(j, 2 * N - 1) d[i][j] = 1;
   ```

2. **标记不可能的移动**：
   - 遍历棋盘所有的`o`位置（表示棋子所在的位置）。
   - 对于每个棋子，检查其相对于空白`'.'`位置的移动向量`(dx, dy)`，并将这些向量标记为不可能的移动。

   ```cpp
   forn(i, N) forn(j, N) {
       if (f[i][j] != 'o') continue;
       forn(x, N) forn(y, N) if (f[x][y] == '.') d[x - i + N - 1][y - j + N - 1] = 0;
   }
   ```

3. **构建模拟攻击范围**：
   - 再次遍历所有`o`位置的棋子。
   - 对每个棋子，检查所有可能的移动向量，如果该向量没有被标记为不可能，计算其攻击的目标位置，并将目标位置标记为被攻击。

   ```cpp
   forn(i, N) forn(j, N) {
       if (f[i][j] != 'o') continue;
       forn(x, 2 * N - 1) forn(y, 2 * N - 1) {
           int xx = i + x - (N - 1), yy = j + y - (N - 1);
           if (min(xx, yy) < 0 || max(xx, yy) >= N || !d[x][y]) continue;
           t[xx][yy] = 1;
       }
   }
   ```

4. **验证模拟结果**：
   - 检查生成的攻击范围是否与初始棋盘上的攻击范围一致。如果一致，则说明找到了合理的移动规则；否则，返回`NO`。

   ```cpp
   bool ok = true;
   forn(i, N) forn(j, N) {
       if (f[i][j] == 'o') continue;
       ok &= (f[i][j] == 'x') == (t[i][j] == 1);
   }
   ```

5. **输出结果**：
   - 如果验证通过，输出`YES`并打印`(2N-1)×(2N-1)`大小的攻击矩阵。
   - 否则输出`NO`。

   ```cpp
   d[N - 1][N - 1] = 2;
   if (ok) {
       cout << "YES\n";
       forn(i, 2 * N - 1) {
           forn(j, 2 * N - 1) cout << ".xo"[d[i][j]];
           cout << '\n';
       }
   } else cout << "NO\n";
   ```

### 复杂度分析
该算法主要通过双重循环遍历棋盘上的所有位置，因此时间复杂度为`O(N^4)`，可以接受。

通过以上步骤，可以有效推导出一个符合题意的棋子移动规则。
